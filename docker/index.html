守护式容器

1. 能够长期运行

2. 没有交互式会话

3. 适合运行应用程序和服务

以守护式运行容器

Ctrl+p Ctrl+q

在 vscode 中无法使用该方法, vscode 将会优先 vscode 自身的快捷键

PS C:\Users\Night\docker> docker run -it ubuntu bash keyboard shortcuts -> Ctrl+p Ctrl+q PS C:\Users\Night\docker> docker ps CONTAINER ID IMAGE COMMAND CREATED STATUS PORTS NAMES 59db504927f4 ubuntu "bash" 9 seconds ago Up 7 seconds hardcore_mayer

附加到运行中的容器

PS C:\Users\Night\docker> docker attach 76b9a4daf21c root@76b9a4daf21c:

启动守护式容器

docker run -d IMAGE [COMMAND] [ARG]

1. -d 以后台的形式启动该进程,但要注意的是:当该命令运行结束后,容器会自动停止

PS C:\Users\Night\docker> docker run --name ubun_dc2 -d ubuntu /bin/sh -c "echo 1" 7aeda0992754f93a301c2eed9fa271324eec0d63c8641fb53fca55d89e04676f

查看容器日志

docker logs [-f] [-t] [--tail] CONTAINER

1. -f --follows=true|false 默认为false 持续跟踪日志的变化,并返回结果

2. -t --timestamps=true|false 默认为false 在返回的结果上,加上时间戳

3. --tail="all" 返回结尾处[ ]条日志,默认返回所有

PS C:\Users\Night\docker> docker logs -ft "docker logs" requires exactly 1 argument. See 'docker logs --help'. Usage: docker logs [OPTIONS] CONTAINER Fetch the logs of a container

->

设置容器的端口映射

run [-P] [-p]

1. [-P] --publish-all=true|false 默认为false

将为容器所有暴露的端口进行映射

docker run -P -i -t ubuntu /bin/bash

2. [-p] --publish=[]

将为指定的端口进行映射

containerPort

docker run -p 80 -it ubuntu /bin/bash

hostPost:containerPort

docker run -p 8080:80 -it ubuntu /bin/bash

ip::containerPort

docker run -p 0.0.0.0:80 -it ubuntu bash

ip:hostPort:containerPort

docker run -p 0.0.0.0:8080:80 -it ubuntu bash
